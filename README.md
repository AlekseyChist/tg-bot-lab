# 🚴 Avala Segment Bot

*бот показывает время участников чата на Strava-сегменте Avala.*

## 🧠 Фишка: «облачный мозг + локальные руки»

В проекте используется гибридный подход к разработке:

*   **Claude (Облако):** Выполняет роль архитектора. Проектирует структуру, дробит задачи на подзадачи и пишет спецификации-промпты в папку `orchestrator/prompts/`. Также Claude запускает тесты и корректирует спецификации при ошибках.
*   **qwen3-coder (Локально):** Вся генерация кода происходит локально на видеокарте RTX 4090 через скрипт `orchestrator/gen.py`, который выступает мостом к Ollama.

**Итог:** Платный AI используется только для планирования и контроля качества, а сама генерация кода бесплатна и выполняется на локальной GPU.

## 🎯 Команды

| Команда | Что делает |
| :--- | :--- |
| `/link` | Привязать свой Strava (один раз, через OAuth) |
| `/board` | Показать таблицу времён всех привязанных участников |
| `/badges` | Повесить время каждому как бейдж рядом с именем (custom title) |
| `/unlink` | Отвязать аккаунт Strava |
| `/help` | Вывести справку по командам |

## ⚠️ Важное про Strava API

Официальный API лидербордов сегментов Strava был удалён в 2020 году — получить «время всех разом» больше нельзя.

Поэтому реализован следующий механизм:
1. Каждый участник один раз привязывает свой аккаунт через OAuth.
2. Бот запрашивает личный рекорд (PR) пользователя из поля `athlete_segment_stats.pr_elapsed_time`.
3. Для работы достаточно прав доступа `scope=read`.

## 🏗 Архитектура

| Файл | Роль |
| :--- | :--- |
| `bot/logic.py` | Чистые функции (формат времени, генерация таблицы, бейджи). Тестируются офлайн. |
| `bot/strava.py` | Async-клиент Strava (OAuth, обновление токенов, запрос PR на сегменте). |
| `bot/storage.py` | Хранилище токенов: Upstash Redis (прод) или JSON-файл (локальный фолбэк). |
| `bot/oauth_server.py` | Обработка OAuth-callback от Strava. |
| `bot/handlers.py` | Обработчики команд для aiogram. |
| `bot/main.py` | Локальный запуск: polling + встроенный OAuth-сервер. |
| `api/index.py` | Единый ASGI-entrypoint для Vercel (вебхук Telegram + OAuth-callback). |
| `scripts/set_webhook.py` | Скрипт для регистрации вебхука Telegram на публичный адрес. |
| `orchestrator/gen.py` | Мост к локальной модели qwen через Ollama. |

> **Принцип:** Чистая бизнес-логика отделена от ввода/вывода (I/O), поэтому её можно тестировать без наличия реальных токенов и подключения к интернету.

## 🚀 Запуск локально (polling)

```bash
python -m venv .venv
# Активация виртуального окружения:
# Windows:
.venv\Scripts\activate
# Linux/macOS:
source .venv/bin/activate

pip install -r requirements.txt
cp .env.example .env
# Заполните .env: BOT_TOKEN (от @BotFather), STRAVA_CLIENT_ID и STRAVA_CLIENT_SECRET (strava.com/settings/api)

python -m bot.main
```

**Примечание:** Локально бот работает, пока включён ваш ПК. Для привязки друзей нужен публичный адрес для OAuth-callback. Можно использовать туннель `cloudflared` к `localhost:8000`. Домен этого туннеля нужно указать в поле *Authorization Callback Domain* в настройках приложения на Strava. Без подключения Upstash хранилище автоматически падает на файл `bot/data/tokens.json`.

## ☁️ Хостинг на Vercel (webhook, всегда онлайн)

В продакшене бот живёт на Vercel в режиме вебхука и не зависит от включения вашего компьютера.

1.  Задеплоить проект на Vercel (`api/index.py` — точка входа, см. `vercel.json`).
2.  В переменных окружения проекта на Vercel задать:
    *   `BOT_TOKEN`
    *   `STRAVA_CLIENT_ID`
    *   `STRAVA_CLIENT_SECRET`
    *   `OAUTH_PUBLIC_BASE` (публичный https-адрес приложения, например `https://<app>.vercel.app`)
3.  Подключить хранилище **Upstash Redis** через Vercel Marketplace. Это создаст переменные `KV_REST_API_URL` и `KV_REST_API_TOKEN`, которые `config.py` читает автоматически (необходимо, так как у serverless-функций нет постоянного диска).
4.  В настройках приложения на [strava.com/settings/api](https://www.strava.com/settings/api) вписать *Authorization Callback Domain* = домен вашего приложения на Vercel (без `https://` и без слеша `/`).
5.  Зарегистрировать вебхук: локально, с заполненными `BOT_TOKEN` и `OAUTH_PUBLIC_BASE`, запустить:

```bash
python scripts/set_webhook.py
```

> **Важно:** Изменения в переменных окружения применяются только после нового деплоя (`vercel --prod`).

## 🔒 Секреты

Локально токены и ключи хранятся только в файле `.env` (а также данные пользователей — в `bot/data/`). Все эти файлы прописаны в `.gitignore`, поэтому они не попадают в репозиторий. В продакшене секреты хранятся исключительно в переменных окружения Vercel. В исходном коде секретов нет.
