# CLAUDE.md — правила работы в этом проекте

## 🔒 ЖЕЛЕЗНОЕ ПРАВИЛО: код пишет ТОЛЬКО локальная LLM

**Claude НЕ пишет прод-код руками. Никогда.**
Весь код бота пишет **локальная qwen3-coder (RTX 4090)** через `orchestrator/gen.py`
(мост к Ollama на `http://localhost:11434`).

Роли жёстко разделены:

| Кто | Что делает |
|---|---|
| **Claude (облако)** | архитектура, план, дробление на подзадачи, спеки-промпты (`orchestrator/prompts/*.md`), запуск `gen.py`, тесты, верификация, git по явной команде |
| **qwen (видяха)** | пишет ВЕСЬ код по спекам — файл за файлом |

**Почему:** платный AI тратится только на мышление и контроль; генерация кода
бесплатна и приватна на локальной GPU. Плюс — неограниченные итерации без оглядки
на счёт.

## Как применять

1. Нашёл баг в сгенерированном коде — **НЕ правь руками**. Уточни спеку в
   `orchestrator/prompts/*.md` и **перегенери файл** через `gen.py`.
2. Исключение: подгонка контракт-тестов под разумный выбор модели (осознанно).
3. Тесты, верификационные скрипты, README/доки — тоже через qwen (не прод-код
   Claude может писать сам только для одноразовой проверки в scratchpad).

## Пример прогона одной подзадачи

```bash
python orchestrator/gen.py \
  --model qwen3-coder-8k:latest \
  --task-file orchestrator/prompts/fN_something.md \
  --out bot/something.py
pytest -q
```

## Git

- Коммит / пуш / создание репозитория — **только по явному разрешению** пользователя.
- Наблюдение («не вижу на гитхабе») — это НЕ команда. Спрашивать.
- Перед публичным репо проверять, что секреты (`.env`, `bot/data/`) в `.gitignore`.

## Секреты

`.env` (BOT_TOKEN, STRAVA_CLIENT_ID/SECRET) и `bot/data/` (токены пользователей) —
в `.gitignore`, в репозиторий не попадают. В коде секретов нет.

## Стиль общения

По-русски, простым языком, без воды.

> Это же правило применяется и в других проектах (например Android Studio): там
> qwen пишет Kotlin/Java, Claude проектирует и пишет спеки. `gen.py` языконезависим.
